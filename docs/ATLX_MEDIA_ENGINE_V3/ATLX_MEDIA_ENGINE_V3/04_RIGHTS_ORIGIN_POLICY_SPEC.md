# 04 - RIGHTS, ORIGIN & POLICY SPEC

## 1. Purpose

This document governs media origin, rights declaration, ownership, demo status, download permissions, watermark decisions and policy evaluation.

This is not legal advice. It is technical governance so Antojados can implement safer media handling.

## 2. Official Origin Types

| origin_type | Meaning |
|---|---|
| official_antojados | Content generated by AntojadosMX employees in assigned functions |
| explorer_partner | Content generated by external explorers/partners under contract/license |
| created_in_antojados | Content created by regular users inside Antojados |
| uploaded_by_owner | Content uploaded by author/business/owner |
| licensed_content | Content with explicit license/authorization |
| external_platform | Content from TikTok/Instagram/Facebook/YouTube/etc. |
| demo_content | Temporary demo/reference content |
| unknown | Unclassified origin |

## 3. Employee-generated Content

If an explorer is an employee and creates content during assigned AntojadosMX functions, the engine must classify it as:

```text
origin_type = official_antojados
employment_generated = 1
ownership_type = company_owned
license_type = employee_generated
engine_watermark_policy = apply
allow_engine_watermark = 1
allow_public_display = 1
```

This requires labor/contract paperwork outside the engine. The engine records the policy result; it does not replace legal documentation.

## 4. Explorer Partner Content

If explorer is not employee:

```text
origin_type = explorer_partner
ownership_type = licensed_to_company
license_type = explorer_partner_generated
```

Watermark and reuse depend on partner contract.

## 5. External Platform Content

Default:

```text
origin_type = external_platform
ownership_type = third_party
license_type = external_unverified
engine_watermark_policy = preserve_external
allow_download = 0
allow_engine_watermark = 0
```

Rules:
- Do not remove external watermark.
- Do not add Antojados watermark by default.
- Do not present as Antojados-owned.
- Prefer low feed priority or demo/restricted usage.

## 6. Demo Content

Default:

```text
origin_type = demo_content
is_demo_content = 1
rights_status = restricted
license_type = demo_only
allow_download = 0
allow_remix = 0
demo_disclaimer = 'Contenido de demostracion'
```

Rules:
- Must show demo label in UI.
- Must be replaceable.
- Must support takedown.

## 7. Watermark Decision Matrix

| Origin | Default Watermark |
|---|---|
| official_antojados | Apply Antojados |
| explorer_partner | Contract-based |
| created_in_antojados | Optional |
| uploaded_by_owner | Optional/permission-based |
| licensed_content | License-based |
| external_platform | Preserve external, skip Antojados |
| demo_content | Preserve external, show demo label |
| unknown | Admin review |

## 8. Download Policy

Downloads are disabled by default except:
- company-owned;
- employee-generated official content;
- authorized owner content;
- approved licensed content.

## 9. Share Policy

Sharing should default to Antojados post URLs, not raw file downloads.

## 10. Policy Evaluation Output

Every media should be evaluated into:

- `can_process`
- `can_publish`
- `can_download`
- `can_share`
- `should_apply_engine_watermark`
- `should_preserve_external_watermark`
- `requires_admin_review`
- `policy_reason`

## 11. Feed Priority Guidance

| Origin | Priority |
|---|---:|
| official_antojados | highest |
| explorer_partner | high |
| created_in_antojados | high |
| uploaded_by_owner | medium/high |
| licensed_content | medium |
| demo_content | low |
| external_platform | low |
| unknown | review/low |
